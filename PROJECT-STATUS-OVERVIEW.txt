8. ARCHITECTURE NOTES (STUDENT DATA)
--------------------------------------------------------------------------------

Student roster is class-group centric:
  - class_group_students  → roster per class group
  - students              → login accounts (phone, password, level sync)
  - Staff manage via Class Groups → Students list → Show/Edit
  - New global directory at /dashboard/students aggregates across groups

Role access summary:

  Role            Class groups    Global students    Edit students
  ------------    -------------   ----------------   -------------
  Super admin     All             All                Yes (in scope = all)
  Coordinator     Scoped          Scoped             Yes (in scope)
  Examiner        Assigned only   No (redirect)      View only + fallback code
  System monitor  Enterprise      Intelligence only  N/A
